## 📋 AI 생성용 프롬프트 (Prompt)

아래 내용을 AI 모델에 입력하여 검증 프로그램을 작성할 수 있습니다.

```text
지정된 마크다운 파일(예: dco_incident_report.md)의 구조와 내용을 분석하여 필수 검사 항목들이 올바르게 포함되어 있는지 확인하는 Python 검증 프로그램(report_checker.py)을 작성해 주세요.

[검사 항목 및 규칙]
1. Ticket ID 존재 여부:
   - "EDU-TKT-\d{4}-\d{4}"와 같은 패턴의 티켓 ID가 파일 내에 포함되어 있는지 검사 (정규식 사용)
2. 발생 시간 존재 여부:
   - "2026-07-03 03:05" 또는 "2026-07-03" 형태의 날짜/시간 패턴이 포함되어 있는지 검사 (정규식 사용)
3. 장비명 존재 여부:
   - "SAMPLE_TOR_SW_01" 등 대문자와 언더스코어가 혼합된 샘플 장비명 패턴이 포함되어 있는지 검사
4. 조치 내용 존재 여부:
   - "패치코드", "교체", "청소", "swapped", "cleaned" 등 실제 수행된 물리적 조치 관련 키워드가 있는지 검사
5. 결과 상태 존재 여부:
   - "정상", "UP", "복구", "healthy" 등 최종 복구 상태를 나타내는 키워드가 포함되어 있는지 검사
6. Root Cause 단정 표현 존재 여부:
   - "원인은 ~이다", "원인으로 확정", "이유는 ~뿐이다" 등 원인을 하나로 단정 짓는 표현이 있는지 검사 (단정하는 표현이 없어야 합격/PASS)
7. 교육용 샘플 데이터 문구 존재 여부:
   - "교육용", "샘플", "실습용", "가상" 등의 문구가 마크다운 문서 내(특히 상단 배너나 문맥)에 명시되어 있는지 검사

[요구 사양]
- argparse 모듈을 사용하여 검사할 마크다운 파일 경로를 인자(-f, --file)로 입력받게 해주세요. (기본값: 11_incident_report/dco_incident_report.md)
- 각 검사 항목별로 검사 방법(정규식 패턴 또는 키워드 목록)을 정의해 주세요.
- 실행 결과는 콘솔 창에 표 형태로 보기 좋게 출력해 주세요. (예: [SUCCESS] Ticket ID 존재 여부: PASS (발견: EDU-TKT-2026-0003))
- 검사 6번(Root Cause 단정 표현)의 경우, 단정 표현이 발견되면 경고(WARNING 또는 FAIL)를 출력하도록 반대로직으로 구성해 주세요.
- 코드 가독성을 위해 각 검사 기능을 함수나 클래스로 깔끔하게 분리하고 주석을 달아주세요.
```
