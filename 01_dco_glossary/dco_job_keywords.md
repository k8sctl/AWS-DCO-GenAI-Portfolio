# [교육용 샘플] DCO 직무 키워드 정의서

*본 문서는 폴리텍대학 AX과정 교육 실습을 위해 가상의 데이터센터 운영 정보를 기초로 작성한 포트폴리오 산출물입니다.*

## 1. DCO 핵심 직무 키워드 5대 요약

1. **하드웨어 트러블슈팅 (Hardware Troubleshooting)**
    - 쉬운 뜻: 컴퓨터 부품인 RAM, SSD, 쿨러 팬이 고장 났을 때 기계를 고치고 부품을 수리하는 행동입니다.
    - 준비 각오: 서버 컴퓨터의 내부 섀시 결합 순서를 꼼꼼히 공부하겠습니다.

2. **네트워크 트러블슈팅 (Network Troubleshooting)**
    - 쉬운 뜻: 서버 간 통신이 끊기거나 느려졌을 때 케이블, 스위치 포트, 링크 상태를 점검해 물리 계층(L1)부터 문제 원인을 찾아 해결하는 업무입니다. 랙 상단의 ToR 스위치와 배선 상태 확인이 대표적인 작업입니다.
    - 준비 각오: OSI 계층별 장애 절차와 스파인-리프 등 데이터센터 네트워크 토폴로지를 학습해, 링크 다운 상황에서 원인을 빠르게 좁혀가는 역량을 갖추겠습니다.

3. **가상화 인프라 구축·트러블슈팅 (Virtualization Build & Troubleshooting)**
    - 쉬운 뜻: 물리 서버 위에 하이퍼바이저를 올려 여러 대의 가상머신(VM)을 운영하는 환경을 구성하고, VM이 안 뜨거나 성능이 떨어질 때 호스트 자원과 설정을 점검해 해결하는 업무입니다. 물리 하드웨어 상태가 가상화 계층의 안정성으로 직결됩니다.
    - 준비 각오: KVM 등 하이퍼바이저를 직접 설치·운영해 보며, 하드웨어 장애가 VM에 미치는 영향을 실습으로 확인하겠습니다.

4. **자산 관리 (Asset Management / Inventory)**
    - 쉬운 뜻: 데이터센터 안의 서버와 부품이 '어디에, 몇 개, 어떤 상태로' 있는지 입고부터 폐기까지 전산으로 기록하고 추적하는 업무입니다.
    - 준비 각오: 스프레드시트 기본 기능을 미리 익히고, 모든 작업을 기록으로 남기는 꼼꼼함을 키우겠습니다.

5. **티켓 기반 업무 처리 (Ticketing System / Work Order)**
    - 쉬운 뜻: 모든 작업을 구두 지시가 아닌 '티켓'이라는 전산 작업 요청서로 접수하고, 처리 과정과 결과를 기록하는 방식입니다. 처리 기한(SLA)을 지키는 것이 핵심입니다.
    - 준비 각오: 작업 내역을 간결하고 정확하게 기록하는 연습을 통해 SLA 준수 역량을 갖추겠습니다.

## 2. 1차시 실습 소감
- 이번 실습에서 생성형 AI 비서와 함께 DCO 직무 키워드를 정리하면서, 평소 알고 있던 데이터센터 업무의 범위를 한층 구체적으로 그려볼 수 있었습니다. 하드웨어 교체나 네트워크 점검 같은 익숙한 작업뿐 아니라, 자산 관리의 전산 추적처럼 세밀한 절차 하나하나가 서비스 신뢰성으로 이어진다는 점이 인상 깊었습니다. 특히 평소 관심이 많던 가상화와 네트워크가 결국 현장의 물리 인프라 위에서 동작하는 만큼, DCO 경험은 인프라 스택 전체를 아래에서부터 이해하는 좋은 출발점이 되겠다고 느꼈습니다. 또한 모든 작업이 티켓으로 기록되고 SLA라는 기한 안에서 관리된다는 점에서, 개인의 성실함과 기록 습관이 곧 서비스 품질로 직결되는 직무라고 생각했습니다. 다음 차시에는 데이터센터 네트워크 토폴로지와 하이퍼바이저 구조를 직접 조사해 정리해보고 싶습니다.
