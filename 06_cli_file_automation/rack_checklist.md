# [교육용 샘플 데이터 / FOR EDUCATIONAL USE ONLY]
> 본 문서는 교육용 실습을 위해 가상으로 작성된 샘플 데이터이며, 실제 인프라, 실제 장비, 실제 고객 정보와는 전혀 무관합니다.

## 교육용 랙 점검 체크리스트 (EDU_RACK_CHECKLIST_SAMPLE)

본 체크리스트는 실제 장비 점검 절차가 아니며, 교육용 샘플 로그와 샘플 티켓을 읽고 기록해야 할 항목 중심으로 구성된 템플릿입니다.

### 1. 기본 정보 기록 및 확인
- [ ] **발생 시간 확인 (Occurrence Time)**
  - 티켓이나 로그에 기록된 가상의 발생 시간을 정확히 확인하였는가? (예: `2026-07-07 06:00:00` 등 샘플 시간 기록)
- [ ] **샘플 장비명 확인 (Sample Device Name)**
  - 영향받은 가상 장비명이 `SAMPLE_` 또는 `EDU_` 접두어 형식으로 올바르게 기재되어 있는지 확인하였는가? (예: `SAMPLE_TOR_SW_01` 확인)
- [ ] **반복 여부 확인 (Recurrence Check)**
  - 동일한 샘플 장비 또는 유사한 교육용 세그먼트에서 이전에도 동일 이벤트(예: CRC error 증가 및 Link Down)가 반복적으로 발생했었는지 이력을 기록하였는가?

### 2. 티켓 및 상태 확인
- [ ] **티켓 상태 확인 (Ticket Status)**
  - 해당 장애 티켓이 현재 어떤 단계(Open / Sample-Investigating / Resolved 등)인지 확인하여 기록하였는가?
- [ ] **원인 단정 금지 원칙 준수 (Do Not Assume Cause)**
  - 확인되지 않은 물리적 불량(케이블 불량, 모듈 불량 등)을 원인으로 단정 짓지 않고, 가상 관찰된 현상(CRC error, Link Down)만을 객관적으로 기록하였는가?

### 3. 후속 조치 및 보고서 작성
- [ ] **보고서 초안 작성 메모 (Report Draft Memo)**
  - 가상의 상황에 대한 점검 결과 요약, 향후 조치 제안(샘플 장비 교체 테스트 계획 등)을 포함한 교육용 보고서 초안이 작성되었는가?
