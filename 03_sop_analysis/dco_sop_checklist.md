---

## 1. 내가 읽은 교육용 샘플 SOP 제목

SAMPLE-DCO-SOP-04: 샘플 Link Down 이벤트 기록 및 전달 절차

---

## 2. SOP의 목적

**내가 이해한 목적:** 

교육용 샘플 로그에서 Link Down 이벤트가 관찰되었을 때, 원인을 추측하거나 단정하지 않고 발생 시간·장비명·반복 여부·티켓 상태 같은 관찰된 사실만을 일관된 절차로 확인·기록하여, Handover Memo나 Incident Report 초안으로 정확하게 전달할 수 있도록 연습하는 것.

---

## 3. SOP 절차 요약

1. 샘플 로그에서 Link Down 이벤트가 표시된 시간을 확인한다.
2. 이벤트에 표시된 샘플 장비명을 확인한다.
3. 같은 장비에서 반복 이벤트가 있는지 샘플 로그 안에서 확인한다.
4. Ticket opened 또는 Ticket escalated 표현이 함께 있는지 확인한다.
5. 영향 범위를 단정하지 말고, 로그에 나타난 관찰 내용만 정리한다.
6. 혼자 판단하기 어려운 내용은 Escalation 필요 여부로 표시한다.
7. 확인한 내용을 Handover Memo 또는 Incident Report 초안에 넣을 수 있게 짧게 정리한다.

---

## 4. 주의사항

- 실제 장비에 접속하지 않습니다.
- 실제 네트워크 진단을 하지 않습니다.
- 실제 AWS 환경에 접속하지 않습니다.
- 실제 티켓 번호, 실제 장비 시리얼, 실제 IP를 사용하지 않습니다.
- AI 답변을 그대로 복사하지 않고 검토합니다.

내가 추가로 주의해야 한다고 생각한 점:

- Link Down만 보고 끝내지 않고 Link Up(복구) 기록이 있는지도 함께 확인해야, 이벤트가 진행 중인지 종료됐는지 구분할 수 있다.
- "반복"의 기준이 사람마다 다를 수 있으므로, 몇 회·몇 분 이내인지 함께 기록해두면 다음 사람이 오해하지 않는다.
- AI가 정리해준 문장에 "~때문에 발생한 것으로 보인다" 같은 원인 추정 표현이 섞여 있지 않은지 제출 전에 한 번 더 검토한다.

- [ ]  AI 답변에서 위험한 표현을 수정했습니다.

## 6. 내가 만든 SOP 체크리스트

**Link Down 이벤트 확인**

- [ ]  샘플 로그에서 Link Down 이벤트가 표시된 시간을 확인했다.
- [ ]  이벤트에 표시된 샘플 장비명(SAMPLE/DEMO/EDU 접두어)을 확인했다.
- [ ]  같은 장비에서 반복된 Link Down 이벤트가 있는지 로그 안에서 확인했다.
- [ ]  Link Up(복구) 기록이 있는지 확인했다.

**티켓 및 전달 여부 확인**

- [ ]  로그에 Ticket opened 표현이 있는지 확인했다.
- [ ]  로그에 Ticket escalated 표현이 있는지 확인했다.
- [ ]  혼자 판단하기 어려운 내용은 "Escalation 필요 여부"로 표시했다.

**기록 정리**

- [ ]  영향 범위를 단정하지 않고, 로그에 나타난 관찰 내용만 정리했다.
- [ ]  원인을 하나로 단정하는 문장이 없는지 검토했다.
- [ ]  확인한 내용을 Handover Memo 또는 Incident Report 초안용으로 짧게 정리했다.

**안전 규칙 준수**

- [ ]  실제 장비에 접속하지 않았다.
- [ ]  실제 네트워크 진단 명령을 실행하지 않았다.
- [ ]  실제 IP 주소, 티켓 번호, 장비 시리얼을 사용하지 않았다.
- [ ]  교육용 샘플 로그에 나온 내용만 근거로 작성했다.
